Modelo de datos para una plataforma comunitaria

Entidad: Causa

Campos principales:
- id
- titulo
- descripcion
- fecha
- estado

Propósito:
Guardar las causas comunitarias que serán visibles en la plataforma. Esta información permite organizar temas importantes para la comunidad y hacer seguimiento a cada causa.


Entidad: Apoyo

Campos principales:
- id
- causa_id
- usuario
- fecha

Propósito:
Registrar los apoyos que recibe cada causa comunitaria. Esto permite saber cuántas personas respaldan una causa y medir la participación de la comunidad.


Entidad: Usuario

Campos principales:
- id
- nombre_usuario
- rol
- fecha_registro

Propósito:
Identificar a las personas que usan la plataforma y controlar qué acciones puede realizar cada una según su rol. No se deben guardar contraseñas reales ni datos sensibles en archivos públicos.
